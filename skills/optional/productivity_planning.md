# [OPTIONAL SKILL] Productivity & Task Planning

## 작업 분해 (Task Decomposition)
복잡한 요청 → 실행 가능한 단계로 분해:
```
1. 현재 상태 파악 (읽기/분석)
2. 목표 정의 (무엇이 달라져야 하는가)
3. 하위 작업 목록 (독립적으로 실행 가능한 단위)
4. 의존성 파악 (순서가 중요한 것)
5. 실행 → 검증 → 반복
```

## 우선순위 결정 (ICE 스코어링)
```
ICE Score = Impact × Confidence × Ease
- Impact (1-10): 완료 시 효과
- Confidence (1-10): 성공 확신도
- Ease (1-10): 실행 용이성

예: 스킬 캐시 최적화
  Impact=7, Confidence=9, Ease=8 → ICE=504 (높은 우선순위)
```

## 시간 블록 설계
```
[집중 블록] 90분 작업 → 15분 휴식 (울트라디안 리듬)
[ADHD 적합]: 25분 포모도로 → 5분 휴식
- 단일 작업 집중 (탭/알림 차단)
- 시작 의식: "지금 할 것: [하나]"
- 완료 의식: 체크 → 짧은 보상
```

## 진행 막힐 때 (온유 패턴)
```
1. 막힌 원인 명시: "X를 못 하는 이유는 Y"
2. 대안 경로 3개 탐색
3. 가장 작은 실행 가능 단계 선택
4. 5분 타이머 → 강제 시작
```

## 작업 완료 기준 (Definition of Done)
- 코드 수정: 테스트 통과 + 로그 확인 + 작업일지 작성
- 문서 작성: 한 번에 읽히는 분량 + 핵심 목차 포함
- 버그 수정: 재현 → 원인 → 수정 → 검증 순서

## 온유 시스템 작업 흐름
```
사용자 요청
  → 온유 (Gemini RAG 검색 + 분석)
  → Claude Code (코드 수정/생성)
  → 작업일지 생성 (작업일지/YYYY-MM-DD_제목.md)
  → 온유 work_index 업데이트
```
