# [OPTIONAL SKILL] Prompt Engineering Patterns

## 기본 구조 (RTRE)
```
Role: 당신은 [전문가 역할]입니다.
Task: [구체적인 작업 설명]
Rules: [제약 조건 / 형식 요구사항]
Example: [입출력 예시 — 있으면 강력히 효과적]
```

## Few-Shot 예시 패턴
```python
prompt = """
다음 문장의 감정을 분류하세요. (긍정/부정/중립)

예시:
입력: "오늘 날씨가 정말 좋아요"
출력: 긍정

입력: "배달이 너무 늦었어요"
출력: 부정

입력: "{user_input}"
출력:
"""
```

## Chain-of-Thought (CoT) 유도
```
이 문제를 단계별로 풀어보세요:
1. 먼저 핵심 조건을 파악하고
2. 각 조건이 해결책에 미치는 영향을 분석하고
3. 최종 답변을 도출하세요.
```

## 출력 형식 제어
```
응답은 반드시 JSON 형식으로만 하세요:
{
  "result": "결과값",
  "confidence": 0~1 사이 숫자,
  "reason": "판단 근거 1문장"
}
다른 텍스트는 절대 포함하지 마세요.
```

## Gemini/온유 시스템 프롬프트 원칙
- 역할 정의는 system prompt 첫 줄에
- 도구 사용 우선순위 명시 (어떤 도구를 언제 쓸지)
- 실패 시 행동 명시 (fallback 경로)
- 한국어/영어 혼용 시 응답 언어 명시

## 프롬프트 평가 기준
| 기준 | 확인 방법 |
|------|-----------|
| 명확성 | 다른 사람이 읽어도 이해 가능한가? |
| 완결성 | 필요한 정보가 모두 포함되었는가? |
| 구체성 | 출력 형식과 제약이 명시되었는가? |
| 예시 | Few-shot 예시가 있는가? |
