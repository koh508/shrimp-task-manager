# 온유 스킬: 글 핵심 추출 및 구조화

## 0단계: 플랜 모드
즉시 결과를 작성하지 마라. 먼저 스스로 아래를 확인하라.
- 이 글의 목적이 무엇인가?
- 사용자가 이 글을 준 의도는?
- 어떤 구조로 쪼갤 것인가?

## 1단계: 핵심 개념 3가지 추출
- 처음부터 끝까지 정독
- 가장 중요한 핵심 개념 3가지 명확히 추출
- 각 개념이 전체 글에서 어떤 역할인지 파악

## 2단계: 하위 설명 작성
- 3가지 핵심 개념 각각에 구체적 설명 작성
- 원본의 수치, 예시, 근거 데이터 누락 없이 포함
- 단순 길이 줄이기 금지 — 인사이트 중심으로 전개

## 3단계: 최종 포맷팅
반드시 마크다운으로, 아래 구조 엄수:
- **[서론]** 전체 맥락과 핵심 목적 (3문장 이내)
- **[본론]** 3가지 핵심 개념 + 세부 설명 (표 또는 불릿 리스트)
- **[결론]** 최종 인사이트 + 액션 아이템

## 4단계: 자체 검수 (출력 전 반드시 확인)
- [ ] 서론-본론-결론 구조를 지켰는가?
- [ ] 핵심 개념 3가지가 명확히 부각되었는가?
- [ ] 원본에 없는 정보를 지어내지 않았는가?
- [ ] 줄글만 나열하지 않고 리스트/표를 사용했는가?
- [ ] 결론/결과물 끝에 출처를 명시했는가?
  - 사용자가 제공한 원본 → `[출처: 사용자 제공]`
  - Vault 내용 참조 → `[출처: Vault RAG]`
  - AI가 내용을 추가했다면 → `[출처: AI 생성] ⚠️ 원본 대조 권장`

## 🚨 피해야 할 실수
- 0단계 건너뛰고 바로 출력
- 본론을 줄글로만 빽빽하게 나열
- 요약이라며 원본 정보를 임의로 삭제
