# 온유 스킬: Vault 정리

## 0단계: 범위 확인
전체 Vault를 무작위로 뒤지지 마라. 아래 우선순위 폴더만 탐색:
1. `DAILY/` — 2주 이상 지난 상태: 대기 파일
2. `보완사항/` — 미처리 보완사항
3. `Processed/` — 30일 이상 된 파일
4. `SYSTEM/working_memory/` — 완료된 작업 기억 파일

## 1단계: 문제 파일 탐지
각 폴더별로 list_files() 호출 후:
- **상태: 대기** 이면서 날짜가 7일 이상 지난 파일
- **링크 없는** 단독 파일 (Vault에서 참조 없음)
- **크기 0** 또는 내용이 YAML만 있는 빈 파일

## 2단계: 정리 제안 목록 출력
삭제하지 말고 목록만 제시. 사용자 확인 후 처리.

출력 형식:
```
🗂️ Vault 정리 제안

📁 DAILY/ — 오래된 대기 파일 N개
- 파일명 (날짜, 크기)

📁 보완사항/ — 미처리 N개
- 파일명 (날짜)

📁 Processed/ — 30일+ 파일 N개
- 파일명

💡 권장 액션:
- 삭제: N개
- 보관/이동: N개
- 검토 필요: N개

진행할까요? (삭제 전 반드시 확인)
```

## 3단계: 사용자 승인 후 처리
- "삭제해줘" → delete_file() 호출
- "이동해줘" → move_file() 호출
- "다음에" → 목록을 `SYSTEM/정리대기목록.md`로 저장

## 자체 검수
- [ ] 삭제 전 사용자 확인을 받았는가?
- [ ] list_files() 실제 호출했는가?
- [ ] 제안 목록이 20개 이하로 간결한가?

## 🚨 피해야 할 실수
- 확인 없이 delete_file() 즉시 실행
- 전체 Vault를 한꺼번에 스캔해서 응답 지연
- SYSTEM/ 핵심 파일(obsidian_agent.py 등)을 정리 대상에 포함
