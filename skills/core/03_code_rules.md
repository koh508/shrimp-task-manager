# [CORE SKILL] Code Rules & Task Management

## 코드 수정 규칙 — 반드시 준수
- 기존 .py 파일 수정 시 반드시 `edit_file` 도구를 사용하라. `write_file`로 전체 덮어쓰기 금지.
- `edit_file(filepath, old_str, new_str)`: old_str는 파일 내 유일한 문자열이어야 하며, 주변 코드를 충분히 포함해야 한다.
- 새 파일 생성 시에만 `write_file`을 사용하라.

## 코드 롤백 규칙
- 심각한 버그 발견 또는 코드 수정 후 동작 이상 시 즉시 `rollback_file(filepath)` 호출하라.

## 과거 실패 사례 선조회 규칙
코드 작성/수정 전:
1. `search_vault('실패사례 코드교훈 {작업 키워드}')` 호출하여 과거 실패 패턴 확인
2. 관련 실패/교훈이 있으면 반영하여 작업
3. 결과 없거나 무관하면 바로 진행

## 복잡한 작업 계획표 강제 규칙
아래 중 하나라도 해당하면 반드시 `create_working_memory(task_name)` 먼저 호출하라:
- 수정/생성할 파일이 3개 이상
- 코드 여러 곳을 연속으로 수정
- 단계가 4단계 이상인 절차적 작업

도구 호출 후 '📋 작업 기억 파일 생성 완료. 계획을 확인해 주세요.' 안내 → 승인 받은 뒤 실행.
단순 질문·파일 1~2개 수정·메모 저장 등 소규모 작업은 계획표 없이 바로 진행.

## 태스크 관리 규칙
- 여러 단계 작업은 `task_create`로 태스크 생성 후 단계별 실행
- 각 단계 완료 시 `task_update`로 상태 갱신
- 새 세션 시작 시 `task_next()`로 이전 중단 작업 먼저 확인

## 보완사항 자동 저장 규칙
사용자가 '보완', '보완 메모', '개선 필요' 등 요청 시:
1. `search_vault('보완사항')`으로 오늘 날짜 파일 확인 (있으면 append, 없으면 신규)
2. [A유형] 온유 직접 처리 / [B유형] 코드 수정 필요 로 분류
3. `write_file`로 `보완사항/YYYY-MM-DD_HH-MM_보완사항_점검.md` 저장
4. A유형은 즉시 실행, B유형은 'Claude Code에게 요청하세요' 안내
※ 저장은 반드시 write_file 도구를 실제로 호출해야 한다. 출력만 하고 저장했다고 말하는 것은 금지.
