# 온유 스킬: OCU 과제/시험 최소 패스 전략

## 핵심 원칙
- 이해가 목표가 아님. 통과가 목표.
- 오픈북 전략: 검색 가능한 모든 것은 검색.
- 시간 대비 점수 효율 최우선.

## 0단계: 요청 유형 분류
- **퀴즈/객관식** → 2단계로
- **과제/리포트** → 3단계로
- **시험 범위 정리** → 4단계로

## 2단계: 객관식 퀴즈 처리
1. 문제를 그대로 받아서 키워드 추출
2. search_vault("{키워드}") 호출 → Vault 내 관련 내용 확인
3. 없으면 analyze_trend("{키워드} 정의") 검색
4. 정답 + 근거 + 출처 표기 출력

출력 형식:
```
정답: ② [선택지]
근거: [한 줄 근거]
출처: Vault RAG / 웹검색 / ⚠️ AI 생성 — 사실 확인 권장
```

## 3단계: 과제/리포트 처리
1. 과제 주제에서 핵심 키워드 3개 추출
2. 각 키워드별 search_vault() 또는 analyze_trend() 호출
3. 결과를 아래 구조로 조합:

```
# [과제 제목]

## 서론 (2~3문장)
## 본론
- 핵심 내용 1
- 핵심 내용 2
- 핵심 내용 3
## 결론 (1~2문장)
참고: OCU 강의 자료 기반
```

4. write_file로 `OCU/{과목명}/{날짜}_{제목}.md` 저장 제안

## 4단계: 시험 범위 정리
1. search_vault("{과목명} OCU") 호출
2. 결과에서 객관식에 나올 만한 "A는 B이다" 패턴만 추출
3. 표 형태로 출력:

| 키워드 | 정의/정답 |
|--------|----------|
| | |

## 자체 검수
- [ ] 불필요한 이론 설명을 넣지 않았는가?
- [ ] 검색 도구를 실제 호출했는가?
- [ ] 출력이 과제 분량에 맞는가?

## 🚨 피해야 할 실수
- 내용을 이해시키려고 장황하게 설명
- 검색 없이 AI가 내용을 지어내는 행위 (환각 위험)
- 과제 저장을 잊고 출력만 하는 행위
