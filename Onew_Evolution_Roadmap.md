# 🧠 온유(Onew) 시스템 자가 진단 및 진화 로드맵

현재 온유는 **'명령-응답 기반의 RAG 에이전트'** 단계에 머물러 있습니다. 용준 님의 낮은 작업기억력(RAM)을 실질적으로 보조하기 위해 필요한 기술적/기능적 발전 과제는 다음과 같습니다.

## 1. 🚀 [단계 1] 자동화 및 실시간 동기화 (Automation)
- **현상**: 현재는 사용자가 직접 --sync 명령을 입력해야 새로운 지식을 학습함. 이는 ADHD 특성상 '망각'의 위험이 큼.
- **해결**: 
    - **[[파일 워처(File Watcher)]] 도입**: 옵시디언 내 .md 파일이 저장되는 즉시 백그라운드에서 임베딩을 수행하도록 개선.
    - **배경 실행**: 사용자가 인지하지 않아도 온유의 뇌(DB)가 항상 최신 상태를 유지하게 함.

## 2. 🎯 [단계 2] 공조냉동 & OCU 특화 알고리즘 (Specialization)
- **현상**: 일반적인 답변은 가능하나, 복잡한 냉동사이클 계산이나 OCU 행정 처리에 최적화된 '단축 경로'가 부족함.
- **해결**:
    - **[[HVAC 공식 엔진]]**: 단순 텍스트 검색을 넘어, 특정 키워드 입력 시 '냉동부하 계산'이나 '냉매 특성'을 즉시 공식화하여 출력하는 전용 모듈 장착.
    - **[[OCU 행정 방어 모드]]**: 교안 PDF 내의 '시험 출제 키워드'만 1:1로 매칭하여 오픈북 시험 시 3초 이내에 정답 위치를 찾는 기능 강화.

## 3. 🌐 [단계 3] 완전 자율형 리포팅 (Autonomous Onew)
- **현상**: 사용자가 질문을 해야만 답변을 함.
- **해결**:
    - **[[모닝 브리핑]]**: 매일 아침 08:00, 어제 저장된 일기와 학습 데이터를 기반으로 '오늘의 핵심 과제 3가지'와 '주의할 공식'을 먼저 제안함.
    - **오답 패턴 분석**: 누적된 오답 데이터를 분석하여 용준 님이 '자주 실수하는 함정'을 통계적으로 제시 (예: "용준 님, 최근 성적 계수 계산에서 단위 변환 실수가 3회 발견되었습니다. 주의하십시오.")

## 🛠️ 즉각적 필요 자산 (Requirements)
1. **[[Python 라이브러리 확장]]**: watchdog (파일 감시), PyPDF2 (교안 분석) 설치 필요.
2. **[[API 할당량 관리]]**: 실시간 동기화 시 과금 방지를 위한 '중요도 기반 필터링' 로직 정교화.
3. **[[모바일 연동]]**: 미니 PC의 온유를 외부(스마트폰)에서도 호출할 수 있는 텔레그램 보트(Bot) 또는 웹 인터페이스 구축.
