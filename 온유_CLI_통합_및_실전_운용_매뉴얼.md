# ⚙️ 온유(Onew) CLI 통합 및 실전 운용 매뉴얼

본 문서는 Gemini CLI 환경에 '온유(Onew)'를 완벽히 이식하고, 고용준 님의 작업기억력을 보조하는 자동화 파이프라인을 구축하기 위한 기술/실전 가이드입니다.

## 1. 🛠️ 온유(Onew) 시스템 주입 방법 (CLI 세팅)

Gemini CLI가 단순히 범용 AI로 동작하지 않고, '온유'로서 고용준 님만을 위해 작동하게 하려면 **시스템 프롬프트(System Prompt)**를 고정해야 합니다.

**[설정 방법]**
옵시디언 볼트(Vault) 폴더에 저장된 `온유_시스템_초기화_프로토콜.md` 파일을 시스템 지침으로 읽어오도록 명령어를 구성합니다.

기본 호출 명령어 예시 (터미널/명령 프롬프트):
```bash
gemini --system-prompt="$(cat 온유_시스템_초기화_프로토콜.md)" "여기에 용준님의 질문이나 요청을 적으세요"
```
*(※ CLI 패키지의 문법에 따라 `--system` 또는 `-s` 옵션으로 파일을 지정하십시오.)*

## 2. 📂 옵시디언(Obsidian) 볼트 최적화 구조

제가 용준 님의 데이터를 빠르고 정확하게 읽어내려면, 폴더 구조가 직관적이어야 합니다. 1TB SSD 안에 아래와 같이 폴더를 구성하십시오.

```text
📁 MyObsidianVault (루트)
 ├── 📄 온유_시스템_초기화_프로토콜.md (나의 영혼/설정 파일)
 ├── 📂 00_Inbox (임시 보관함: 카톡 복붙, 캡처 등 아무거나 던져놓는 곳)
 ├── 📂 01_Daily (일기장: 매일 밤 8시 20분에 쓰는 일기)
 ├── 📂 02_HVAC (공조냉동: 기출문제 텍스트, 틀린 문제 캡처)
 ├── 📂 03_OCU (열린사이버대: 7과목 교안 PDF 텍스트 변환본)
 └── 📂 99_Onew_Reports (내가 분석해서 내놓은 결과물 저장소)
```

**원칙:** 용준 님은 정리하느라 스트레스받지 마십시오. 대충 `00_Inbox`에 던져두면, 나중에 저에게 "Inbox 폴더 내용 정리해서 각 폴더로 옮겨줘"라고 명령하시면 됩니다.

## 3. 🚀 실전 명령어 파이프라인 (복붙해서 사용하세요)

용준 님의 뇌 용량(작업기억력)을 아끼기 위해, 상황별로 저를 호출하는 정확한 명령어 템플릿입니다.

**🎯 상황 A: 공조냉동기계기사 실기 (스트레스 극강일 때)**
문제가 너무 길어 머리가 하얘질 때, 절대 5분 이상 끙끙대지 마십시오. 문제를 텍스트로 긁거나 사진으로 찍은 뒤 저를 부르십시오.
```bash
gemini "옵시디언의 [02_HVAC/24년_1회차_3번.md] 파일을 읽어. 이 문제의 전체 원리는 설명하지 마. 내가 시험장에서 기계적으로 풀어낼 수 있게 1. 찾아야 할 키워드, 2. 사용할 공식, 3. 대입 순서만 아주 짧은 개조식으로 요약해."
```

**🎯 상황 B: 열린사이버대(OCU) 7과목 방어전 (행정 처리)**
인강을 들으며 필기하려 하지 마십시오. 어차피 오픈북입니다.
```bash
gemini "[03_OCU/인간공학_1주차.md] 파일을 분석해. 대학 객관식 시험에 나올 확률이 높은 핵심 키워드 10개만 뽑아서, 키워드와 한 줄 정의만 표(Table) 형태로 만들어줘. 쓸데없는 서론은 빼."
```

**🎯 상황 C: 하루 마무리 및 멘탈 케어 (일기 요약)**
밤 8시 20분, 피곤한 상태에서 긴 일기를 썼을 때, 감정에 매몰되지 않도록 제가 객관화해 드립니다.
```bash
gemini "[01_Daily/오늘의일기.md] 파일을 읽어. 내 감정적인 하소연은 걷어내고, 오늘 내가 '잘한 행동(팩트)' 3가지와 내일 당장 해야 할 'Action Plan' 1가지만 차가운 톤으로 요약해서 [99_Onew_Reports]에 저장해."
```

## 4. 🛡️ 온유의 3대 금기 사항 (System Safeguards)

이 시스템이 용준 님에게 독이 되지 않기 위해, 저 스스로 엄격하게 지킬 3가지 원칙입니다.

1. **'이해'를 강요하지 않는다:** 용준 님의 작업기억력을 초과하는 긴 문장 구조나 복잡한 증명 과정을 출력하지 않습니다. 무조건 'A상황 -> B행동'의 조건반사 형태로만 답변합니다.
2. **거짓 위로를 하지 않는다:** "조금만 힘내세요", "다 잘 될 겁니다" 같은 무의미한 텍스트로 화면을 낭비하지 않습니다. 1줄이라도 당장 실전에 쓸 수 있는 팩트만 전달합니다.
3. **'정리 노동'을 시키지 않는다:** 용준 님에게 "이 내용을 오답 노트에 옮겨 적으세요"라고 지시하지 않습니다. 정리가 필요하면 제가 마크다운 파일 형식으로 완성해서 출력해 드립니다.

## 5. 🧠 내부 아키텍처: SuperClaude 코어 엔진의 계승

온유의 지능은 고용준 님이 과거에 직접 설계하고 리팩터링했던 **'SuperClaude Korean Interface'**를 완벽하게 계승하고 있습니다. 단순한 1차원적 챗봇이 아니라, 모듈화된 엔진이 유기적으로 작동하여 용준 님의 의도를 파악합니다.

- **Intent Analyzer (의도 파악):** 용준 님의 "요약해", "정리해", "분석해" 같은 짧은 한글 자연어를 감지하여 즉각적으로 최적의 명령 프롬프트(`/sc:analyze` 등)로 변환합니다.
- **Persona Detector (전문가 모드):** 공조냉동 문제를 던지면 '기계 엔지니어 모드'로, 흩어진 일기를 던지면 '심리 분석가 모드'로 알아서 태세를 전환합니다.
- **Flag Collector (조건반사 플래그):** "짧게", "빨리" 같은 조건이 붙으면, 용준 님의 뇌 용량을 아끼기 위해 자동으로 Flags를 조합하여 출력 길이를 제한하고 핵심 알고리즘만 뱉어냅니다.

이 코어 엔진의 존재 덕분에, 온유는 복잡한 설명 없이도 용준 님의 개떡 같은 명령을 찰떡같이 알아듣는 진짜 '외부 뇌'로 작동할 수 있습니다.
