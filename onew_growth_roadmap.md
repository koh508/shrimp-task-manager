# 온유 시스템 성장 로드맵

> **작성 원칙:** 현재 코드를 직접 읽고 확인된 것만 기록. 이상적 설계가 아닌 실제 가능한 개선만 포함.
> 기존 `onew_system_gaps_initial_draft.md`는 **참고 전용** — Kafka/K8s 등 과도한 제안은 채택 안 함.

---

## 현재 시스템 실제 수준

| 항목 | 실제 상태 |
|------|-----------|
| 언어/환경 | Python 3.9+, Windows 로컬 실행 |
| 핵심 AI | Gemini 2.5-flash (대화/요약), gemini-embedding-001 |
| 벡터 DB | LanceDB (3072차원, 코사인 유사도) |
| 파일 감시 | watchdog |
| 안정성 점수 | 핵심기능 85% / 부가기능 60% |

**잘 돌아가는 것:** RAG 대화, 도구 자동 실행, 파일 작업, 퀴즈, 캘린더, 시크릿 모드
**불안정한 것:** 야간학습, ADHD 코치, 자동 클리핑 실제 수집, 음성 인식

---

## 개선 우선순위 (3단계)

### 🔴 1단계 — 지금 당장 (안전/과금 관련)

#### 1-1. Embedding API 실제 차단 없음
- **현황:** `MAX_DAILY_EMBED_CALLS = 10000` 설정만 있고, 실제로 초과해도 안 막음
- **개선:** `sync()` 진입 시 오늘 embedding 호출수 체크 → 초과 시 차단
- **난이도:** 낮음 (5줄 수정)
- **파일:** `obsidian_agent.py` — `OnewPureMemory.sync()` 내부

#### 1-2. 동시 쓰기 경합 (lock 미적용 파일)
- **현황:** `onew_location.json`, `User_Profile.md` 등 직접 `open(..., 'w')` 사용
- **개선:** `_atomic_json_write()` / `_atomic_md_append()` 로 교체
- **난이도:** 낮음 (검색→교체)
- **파일:** `obsidian_agent.py` 전반

#### 1-3. 세션 리셋 후 엔티티 누락
- **현황:** 세션 리셋 시 엔티티 워킹메모리가 시스템 프롬프트에 주입되나, User_Profile 업데이트와 타이밍 충돌 가능
- **개선:** `_update_user_profile_facts()` 실패 시 silent pass 확인 (이미 있음 ✅)
- **난이도:** 없음 (오늘 이미 수정됨)

---

### 🟡 2단계 — 이번 달 (신뢰도 향상)

#### 2-1. 야간학습 실제 작동 구현
- **현황:** 폴더 구조·분류 로직만 있고 실제 콘텐츠 수집 미구현
- **개선 방향:**
  - `quiz_me()` file_path 개선 완료 (오늘 ✅) → 야간학습 폴더와 연결
  - 실기 합본 파트 파일들을 야간학습 '공조냉동' 카테고리로 자동 연결
- **난이도:** 중간
- **파일:** `onew_night_study.py`

#### 2-2. RAG 검색 품질 개선
- **현황:** 임계값 0.15 고정, 검색 결과가 너무 많거나 너무 없는 경우 있음
- **개선 방향:**
  - 질문 유형별 임계값 조정 (일상 질문 0.2 / 전문 검색 0.1)
  - 검색 실패 시 "Vault에 관련 내용 없음" 명확히 표시 (현재는 그냥 AI 생성으로 넘어감)
- **난이도:** 중간
- **파일:** `obsidian_agent.py` — `OnewPureMemory.search()`, `ask()`

#### 2-3. 보완사항 자동 분류 정확도 향상
- **현황:** `onew_meta.py`가 A/B 유형 분류를 자동 생성하는데 엉뚱한 분류 발생
- **개선 방향:**
  - B유형(코드 수정) 판단 기준 명확화
  - 코드 수정 제안 시 실제 함수명/파일명 명시 강제
- **난이도:** 낮음 (프롬프트 수정)
- **파일:** `onew_meta.py`

#### 2-4. ADHD 코치 신뢰도
- **현황:** 과집중 감지, 스트릭 기록 초기 구현 상태
- **개선 방향:**
  - 공부 시간 추적 정확도 확인
  - 의미 없는 알림 줄이기 (현재 너무 자주 또는 부정확하게 트리거)
- **난이도:** 중간
- **파일:** `onew_adhd.py`, `onew_adhd_coach.py`

---

### 🟢 3단계 — 다음 달 이후 (기능 확장)

#### 3-1. 자동 클리핑 실제 수집 구현
- **현황:** 주제 설정만 되고 실제 웹 다운로드 미구현
- **개선:** `fetch_url_as_md()` 기반으로 주제별 검색 → 클리핑 자동 저장
- **난이도:** 중간~높음
- **파일:** `obsidian_agent.py` — `AutoClipper`

#### 3-2. 보완사항 → 작업 자동 연결
- **현황:** 보완사항.md 파일은 쌓이는데 자동으로 처리되지 않음
- **개선:** B유형 항목을 온유 태스크(task_create)로 자동 등록
- **난이도:** 중간
- **파일:** `onew_meta.py` + `onew_task_manager.py`

#### 3-3. 모노리스 분리 (장기)
- **현황:** `obsidian_agent.py` 5000줄+ 단일 파일
- **개선 방향:** 기능별 모듈 분리 (메모리 모듈, 대화 모듈, 도구 모듈)
- ⚠️ **위험도 높음** — 전체 리팩토링이므로 단계적으로만 진행
- **난이도:** 높음

## 채택하지 않는 것 (이유 포함)

| 제안 | 채택 안 하는 이유 |
|------|-----------------|
| Kafka / RabbitMQ | 단일 사용자 로컬 앱에 메시지 큐 불필요 |
| Kubernetes / Docker | 로컬 Windows 실행 환경, 컨테이너화 이득 없음 |
| PostgreSQL / SQLAlchemy | LanceDB로 충분, ORM 오버엔지니어링 |
| ELK Stack / Prometheus | 개인 사용자 모니터링에 과도, 텔레그램 알림으로 충분 |
| CI/CD 파이프라인 | 혼자 개발·사용하는 시스템에 불필요 |
| asyncio 전환 | threading 기반 현재 구조 안정적, 전환 비용 > 이득 |
