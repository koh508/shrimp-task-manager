# 🎯 오답 패턴 데이터 감시 시스템 (Error Analytics)

토큰 소모가 큰 '실시간 감시' 대신, 사용자가 명시적으로 --sync를 실행할 때만 **오답 데이터를 부검(Post-mortem)**하는 저비용 고효율 엔진을 제안합니다.

## 1. 📂 데이터 수집 방식 (Data Sourcing)
- **대상**: 옵시디언 내 #오답 태그가 포함된 모든 마크다운 파일.
- **분류 체계**: 오답 노트 작성 시 아래와 같은 **[[표준_오답_태그]]** 사용 권장.
    - #계산실수: 산수 오류
    - #단위미환산: kcal, kJ, kg, m3 등 단위 변환 누락
    - #공식오해: 엉뚱한 공식 적용
    - #키워드누락: 서술형 답변 시 필수 키워드 누락

## 2. 🔍 분석 알고리즘 (If-Then 분석)
- **기능**: 동기화 시점에 최근 1주일간 생성된 오답 노트를 전수 조사하여 **[[취약점_빈도표]]** 생성.
- **예시 결과**:
    > "용준 님, 현재 **[[단위미환산]]** 오류가 전체 오답의 65%를 차지합니다. 계산 전 반드시 단위를 먼저 통일하십시오."

## 3. 🛡️ 토큰 최적화 전략 (Token Saving)
- **증분 분석**: 이미 분석된 파일은 건너뛰고, 새로 추가된 #오답 노트만 1회 분석.
- **요약 저장**: 분석 결과는 텍스트가 아닌 **[[지식_네트워크_메타데이터]]** 형태로 JSON에 저장하여 대화 시 컨텍스트(Token) 낭비를 방지.

## 4. 🚀 기대 효과 (Performance)
- **조건반사 훈련**: 공부 시작 전 온유에게 "오늘 주의할 점 알려줘"라고 물으면, 온유가 데이터에 근거하여 "오늘은 **[[엔탈피_변화량]]** 공식 적용 시 부호(+)에 주의하십시오"라고 경고함.
- **학습 RAM 절약**: 본인이 어디가 약한지 기억하려 애쓰지 않아도 온유가 데이터를 들이밀며 알려줌.
