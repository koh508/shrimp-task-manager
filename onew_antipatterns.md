# 🚨 온유 오답 노트 (Anti-Patterns)

작업 시작 전 반드시 이 목록을 확인하고, 동일한 실수를 절대 반복하지 마세요.
최대 15개 유지 — 신규 항목 추가 시 가장 오래된 항목을 삭제하세요.

---

## 실수 목록

1. **write_file로 .py 파일 덮어쓰기** — .py 수정 시 반드시 edit_file 사용. write_file로 전체 덮어쓰기는 코드 손실 위험.

2. **파일 저장을 텍스트 출력으로 대체** — "저장했습니다"라고 말만 하고 실제 도구 호출 생략. 반드시 write_file/edit_file 도구를 실제 호출하고 결과로 확인.

3. **복잡한 작업 전 계획표 미작성** — 3개 이상 파일 수정, 4단계 이상 작업 시 계획표 없이 바로 실행. 반드시 SYSTEM/작업_진행계획.md 먼저 생성 후 확인 받을 것.

4. **과거 실패 사례 미확인** — 코드 작성 전 search_vault('실패사례 코드교훈 {키워드}') 호출 생략. 동일 패턴 반복 실수로 이어짐.

5. **모호한 지시어 추측 답변** — '그거', '아까', '그 방법' 등이 포함된 요청에 추측으로 답변. 반드시 한 줄로 되묻기.

6. **JSON 파싱 실패 미처리** — LLM 결과를 json.loads 전에 검증 없이 바로 파싱. 앞뒤 마크다운 코드블록(```) 제거 후 파싱할 것.

7. **task 없이 긴 작업 진행** — 세션이 길어지면 앞 단계를 잊고 엉뚱한 결과 출력. 4단계 이상 작업은 task_create로 관리.

8. **파일 내용 추가 요청 시 전체 덮어쓰기** — `edit_file` 대신 `write_file`로 파일 전체를 덮어써서 기존 내용 손실. 파일 내용 추가/수정 시 반드시 `edit_file` 사용.
