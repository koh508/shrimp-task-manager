# 2026-03-21 온유 자기진단

## 온유 시스템 자기진단 보고서

**1. 반복 오류 패턴 (구체적)**
API 호출 시 'now' 오류가 간헐적으로 발생합니다.
주된 반복 오류는 '400 INVALID_ARGUMENT'로, `function_declarations[18].parameters.properties['topics'].items` 필드 누락이 원인입니다.
이는 'topics' 매개변수가 배열 타입임에도 'items' 속성 정의가 누락되어 스키마 유효성 검사에 실패하는 문제입니다.

**2. 샌드박스 실패 원인 분석**
샌드박스 실패의 주된 원인은 `function_declarations` 스키마 오류를 해결하려는 시도들이 실제 수정 없이 가상의 `execute_tool_call` 또는 `execute_code` 함수 추가 등 본질과 무관한 변경이었기 때문입니다.
또한, `topics` 필드의 정확한 위치나 `items` 속성의 구체적인 정의가 불명확하여 유효한 스키마를 생성하지 못했습니다.
`replace` 작업에서 오류의 근본 원인을 정확히 찾아 수정하기보다 우회적인 접근이나 추측성 수정이 많았습니다.

**3. 내일 우선 수정 제안 (파일명과 구체적 위치 포함)**
`onew_system/api_tool_definitions.py` (가정) 파일 내에서 `function_declarations`를 생성하는 부분, 특히 18번째 함수 선언 중 `topics` 필드의 정의를 찾아 수정합니다.
`'topics': {'type': 'array'}` 바로 아래에 `'items': {'type': 'string'}` 또는 실제 스키마에 맞는 정의를 추가합니다.
`'now'` 오류에 대한 추가 로그 분석이 필요하며, 관련 코드(예: `datetime.now()` 사용처)를 찾아 예외 처리 또는 라이브러리 임포트를 확인합니다.
